# Phase 02 Plan 09: Cron and Archive Summary

GitHub Actions cron (every 15 min) calls /api/ingest; daily archive cron (03:00 BRT) calls /api/archive which copies the latest snapshot_cache row into a new snapshot_archive table and prunes rows older than 30 days. Both workflows are curl-only and gated on a shared INGEST_TOKEN secret.

## What was built

### Task 1: Cron + Archive Workflows (commit 357a1ce)

- **`.github/workflows/cron.yml`** — schedule `*/15 * * * *` UTC + workflow_dispatch. Single-step curl POST to `${VERCEL_URL}/api/ingest` with Bearer token, `--retry 3 --retry-delay 5 --retry-all-errors --max-time 60`, `timeout-minutes: 5`. No checkout, no setup-node — pure curl.
- **`.github/workflows/archive.yml`** — schedule `0 6 * * *` UTC (= 03:00 America/Sao_Paulo, BRT -03:00) + workflow_dispatch. Same curl pattern targeting `/api/archive`.

Both workflows use the existing `INGEST_TOKEN` and `VERCEL_URL` repo secrets. Failures of one do not block the other (independent workflow files, independent runs).

### Task 2: snapshot_archive table + /api/archive route + tests (commit b7e82c4)

- **`src/db/schema.ts`** (modified) — appended `snapshotArchive` `pgTable` with columns `date date`, `snapshot_key text`, `body jsonb`, `formula_version text`, composite PK `(date, snapshot_key)`. Added `SnapshotArchiveRow` and `NewSnapshotArchiveRow` inferred types.
- **`drizzle/migrations/0001_reflective_tiger_shark.sql`** — auto-generated by `pnpm db:generate`, contains only `CREATE TABLE snapshot_archive` (does not touch existing tables from 0000). `pnpm db:check` confirms schema/migration sync.
- **`src/app/api/archive/route.ts`** — Node-runtime POST handler:
  1. `verifyBearerToken` (constant-time, reused from 02-08)
  2. SELECT latest `snapshot_cache` row by `computedAt desc limit 1`; if empty, return `{ ok, archived: 0, pruned: 0 }`
  3. INSERT into `snapshot_archive` keyed by today's UTC date with `onConflictDoUpdate` for same-day idempotency
  4. DELETE from `snapshot_archive` where `date < (today - 30 days)`, returning pruned count
  5. Return `{ ok: true, archived: 1, pruned, durationMs }`
- **`src/app/api/archive/route.test.ts`** — 5 integration tests gated on `DATABASE_URL_TEST` (lazy `await import` pattern matching plan 02-08 ingest tests so suite skips cleanly when no test DB is available):
  - rejects without auth → 401
  - produces archive row dated today on first call
  - idempotent on second call same day (ON CONFLICT DO UPDATE)
  - prunes archive rows older than 30 days
  - returns archived=0 when snapshot_cache is empty

## Deviations from plan

### Auto-fixed issues

**1. [Rule 1 - Bug] Test file crashed at module load when DATABASE_URL absent**

- **Found during:** Task 2 verification (`pnpm test`)
- **Issue:** Initial test file used static top-level `import { db } from "@/db/node"`, which evaluates `getPool()` at import time and throws `"DATABASE_URL is not set (node)"` before vitest can apply `describe.skipIf(skip)`. Caused suite-level `FAIL` in CI without DB.
- **Fix:** Refactored to lazy-import pattern (matches `src/app/api/ingest/route.test.ts` from plan 02-08): module-level `let dbMod / schemaMod / routeMod` holders populated inside `beforeAll` which only runs when the suite is not skipped.
- **Files modified:** `src/app/api/archive/route.test.ts`
- **Commit:** b7e82c4 (folded into Task 2 commit)

**2. [Spec correction] Migration filename `0001` not `0002`**

- **Found during:** Task 2 step 2
- **Issue:** Plan body's acceptance criteria globbed `drizzle/migrations/0002_*.sql`, but plan 02-02 produced only one prior migration (`0000_majestic_marvex`), so drizzle-kit's next sequential number is `0001`, not `0002`. The user prompt explicitly clarified this ("New SQL file numbered 0001 (sequential after 02-02's 0000)").
- **Fix:** Used the actual emitted filename `0001_reflective_tiger_shark.sql`. Substantive content unchanged: snapshot_archive CREATE TABLE with composite PK matches the spec.
- **Commit:** b7e82c4
