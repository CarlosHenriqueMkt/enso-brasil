# Requirements — ENSO Brasil v1

> v1 = the smallest thing that proves the concept and already has real public value: **a per-state hazard dashboard, in PT-BR, aggregating ≥2 official Brazilian sources via API, fully accessible.** Everything else is a later milestone.

REQ-IDs are atomic and testable. Each maps to exactly one phase in `ROADMAP.md`.

---

## v1 Requirements

### Foundation & Quality (FOUND)

- [ ] **FOUND-01**: Public GitHub repo with MIT LICENSE, README (PT-BR primary, EN secondary), CONTRIBUTING.md, CODE_OF_CONDUCT.md from first commit
- [ ] **FOUND-02**: Next.js 15.5.x App Router scaffolding with TypeScript strict, Tailwind v4, ESLint, Prettier, Husky pre-commit
- [ ] ~~**FOUND-03**~~: REMOVED. Project is PT-BR only; no `next-intl`, no locale routing. Decision reversed during Phase 1 SPEC.
- [ ] **FOUND-04**: GitHub Actions CI runs typecheck, lint, Vitest, Playwright on every PR; caching enabled
- [ ] **FOUND-05**: Mandatory disclaimer SSR-rendered in root layout (not client JS) — visible on every page even with JS disabled. Disclaimer must include all three emergency contacts **paired with agency names**: **199 Defesa Civil · 193 Bombeiros · 190 Polícia**. Never display the bare numbers — the number alone is incomplete information
- [ ] **FOUND-06**: PT-BR privacy/LGPD page at `/privacidade` describing server logs, IP retention, no analytics-by-default
- [ ] **FOUND-07**: Accessibility shell — color-blind safe palette (icon + text + color), keyboard nav baseline, lang="pt-BR" set, focus-visible styles
- [ ] **FOUND-08**: Stale-data and source-failure notices render at the **top of the page** (above all main content), SSR-rendered, never inline-only. Pattern: full-width banner on degraded snapshot health
- [ ] **FOUND-09**: Edge-state copy strings centralized in `src/lib/messages.ts` (plain TypeScript constants — NOT an i18n catalog, since the project is PT-BR only):
  - Verde: "Não encontramos nenhuma emergência nessa localidade. Verifique em outras fontes de informação antes de decidir o que você vai fazer."
  - Stale source: "Não estamos recebendo dados do(a) {fonte}. Acesse {url} diretamente e busque a informação que você precisa."
- [ ] **FOUND-10**: Every link to an official source renders the domain in monospace font (helps users recognize external/official destinations)

### Data Foundation (DATA)

- [ ] **DATA-01**: Drizzle ORM + Neon Postgres schema for `alerts`, `sources_health`, `snapshot_cache` tables with indexes
- [ ] **DATA-02**: Upstash Redis client wired (`@upstash/redis`) for hot-path snapshot cache
- [ ] **DATA-03**: `ofetch` HTTP wrapper with retry/timeout/status-aware retry as the only outbound HTTP client
- [ ] **DATA-04**: `SourceAdapter` interface + registry pattern — adding a 3rd source requires zero changes to orchestrator/risk engine/UI
- [ ] **DATA-05**: GitHub Actions workflow runs every 15 min, calls token-protected `/api/ingest` Vercel endpoint
- [ ] **DATA-06**: `/api/ingest` orchestrates parallel adapter fetches → normalize to `Alert[]` → dedup → persist raw to Postgres → recompute snapshot → write to Upstash → trigger `revalidatePath` for any state whose level changed
- [ ] **DATA-07**: Source-failure tracking: `sources_health` table records last-success timestamp per source; UI surfaces staleness ≥30 min
- [ ] **DATA-08**: Schema validation on every poll response (zod); payload-hash anomaly detection logs when source schema drifts
- [ ] **DATA-09**: Daily Postgres archive of full snapshot (cheap, valuable for M10 historical comparison)

### Risk Engine (RISK)

- [ ] **RISK-01**: Pure function `calculateRiskLevel(alerts: Alert[]): RiskLevel` — edge-safe, no I/O, 100% test coverage
- [ ] **RISK-02**: Risk levels: `green | yellow | orange | red | unknown` (NOT just 4 — `unknown` is mandatory)
- [ ] **RISK-03**: Severity types: `low | moderate | high | extreme`
- [ ] **RISK-04**: Default severity for unknown source terms = `moderate` (conservative bias)
- [ ] **RISK-05**: Deduplication rule: same `hazardType` + same `state` + overlapping time windows = 1 alert for calc, both sources displayed in UI
- [ ] **RISK-06**: Validity window: alerts without `activeUntil` expire after 24h from `publishedAt`
- [ ] **RISK-07**: When all integrated sources are stale > 1h, state level = `unknown` (gray "Dados indisponíveis"), never green-by-default
- [ ] **RISK-08**: Snapshot output is versioned (`formula_version: "v0"`) and additive-only — forward-compatible for M8 public API and M11 notification diffs
- [ ] **RISK-09**: Per-state plain-language explanation generated by the engine ("Laranja: 1 alerta de Perigo do INMET para chuva forte")
- [ ] **RISK-10**: Source severity mapping files: `src/lib/risk/sources/{cemaden,inmet}.ts` with full v0.1 mapping tables

### Source Adapters (ADAPT)

- [ ] **ADAPT-01**: CEMADEN adapter — fetches from Painel de Alertas backing endpoint (undocumented; treat as unstable), schema-validates, normalizes to `Alert[]`, attributes correctly
- [ ] **ADAPT-02**: INMET adapter — parses Alert-AS **CAP XML** (NOT JSON), normalizes to `Alert[]`, attributes correctly
- [ ] **ADAPT-03**: Optional 3rd source in v1 (P5): INPE Queimadas OR NASA FIRMS — country-level single call for FIRMS, NEVER per-state fan-out
- [ ] **ADAPT-04**: Each adapter ships with golden-file fixtures of real source responses + a contract test that fails when the source changes shape

### Dashboard UI (DASH)

- [ ] **DASH-01**: Single home page `/` shows all 27 Brazilian states with current risk level
- [ ] **DASH-02**: Desktop layout — interactive Brazil map (Albers conic, parallels [-7,-22], rotate [54,0]) on right, info panel on left; clicking a state updates the panel
- [ ] **DASH-03**: Mobile layout — vertically stacked per-state cards with search/filter; map appears below as secondary
- [ ] **DASH-04**: Per-state deep-linkable URLs (`/estado/{uf}`) with Open Graph + Twitter card metadata for WhatsApp sharing
- [ ] **DASH-05**: Risk level badge on each state shows: color + icon (✓/⚠/⚠⚠/⛔/?) + textual label ("Sem alertas" / "Atenção" / "Alerta" / "Perigo" / "Dados indisponíveis")
- [ ] **DASH-06**: Each state row shows: plain-language explanation, list of active alerts with source attribution + link to original publication, last-update timestamp ("Atualizado há 8 minutos")
- [ ] **DASH-07**: Region filter chips: Norte / Nordeste / Centro-Oeste / Sudeste / Sul
- [ ] **DASH-08**: "Compartilhar" button per state — copies `/estado/{uf}` URL with prefilled WhatsApp share intent
- [ ] **DASH-09**: "Como calculamos isso?" link on each state row → GitHub README anchor explaining v0 formula
- [ ] **DASH-10**: Yellow color contrast: use darkened yellow OR enforce black text on yellow background (NEVER white text on yellow — WCAG AA fails)

### Accessibility & Performance (A11Y)

- [ ] **A11Y-01**: WCAG AA verified via axe-core in CI on home + at least 3 state pages; zero criticals
- [ ] **A11Y-02**: Keyboard navigation works for map + cards; tab order logical; focus visible
- [ ] **A11Y-03**: `/texto` accessible text-only route — full data, no map, server-rendered, works on 3G + screen readers + JS-off
- [ ] **A11Y-04**: Color-blind safety: every risk level identified by icon + text + color (color is third)
- [ ] **A11Y-05**: Lighthouse perf ≥ 90 on a simulated 3G profile; LCP < 2.5s; total transfer < 200 KB on initial route
- [ ] **A11Y-06**: Screen-reader live region announces state-level changes when user clicks a state

### Deploy & Launch (DEPLOY)

- [ ] **DEPLOY-01**: Site live on Vercel public domain (subdomain or cheap .com.br)
- [ ] **DEPLOY-02**: Environment secrets managed via Vercel + GitHub Actions secrets; documented `.env.example`; pre-commit secret scan
- [ ] **DEPLOY-03**: Health endpoint `/api/health` returns ingestion status + per-source last-success timestamps
- [ ] **DEPLOY-04**: README final pass — quickstart, contributing, fórmula explained, sources listed, known limitations from `risk-formula-v0.md` Risks section
- [ ] **DEPLOY-05**: Plausible analytics integrated (privacy-first), opt-out documented
- [ ] **DEPLOY-06**: Outreach packet to CEMADEN + INMET + Defesa Civil announcing the project, asking for stable endpoint feedback

## Out of Scope — never

- User accounts, social, comments, user-submitted reports
- Forecasting model (we aggregate, we don't predict)
- Affiliate links, shopping, marketplace, brand recommendations
- Monetization of any form
- Replacement of Defesa Civil / CEMADEN as official alert systems

## Definition of Done — v1

1. Dashboard live at a public URL with map (desktop) and cards (mobile) showing all 27 states
2. ≥ 2 official sources integrated via API (CEMADEN + INMET minimum)
3. Risk level computed via versioned v0 formula including `unknown` level
4. Disclaimer SSR-rendered on every page; LGPD page at `/privacidade`
5. Source attribution + timestamp on every shown hazard
6. WCAG AA verified; `/texto` route works
7. 3G performance budget met
8. Public GitHub repo with full OSS scaffolding (LICENSE, README PT-BR+EN, CONTRIBUTING, CODE_OF_CONDUCT)
9. CI green on every PR
10. Site available + monitored; outreach to authorities sent
